# متابع اللوحات

تطبيق ويب (يعمل زي تطبيق موبايل) لمتابعة سياراتك بأرقام اللوحات، مع بحث صوتي بالعربي وتحديد موقع كل سيارة على الخريطة.

## المميزات
- إضافة أكتر من ١٢ سيارة (رقم اللوحة حقل مطلوب).
- بحث صوتي بالعربي: تتكلم وهو يكتب الكلام في مربع البحث لحظة بلحظة، ويقولك بصوت عربي "موجودة" أو "مش موجودة".
- **بحث بالكاميرا (OCR)**: تصوّر لوحة سيارة بكاميرا الموبايل، والتطبيق يقرأ الحروف والأرقام العربية منها تلقائيًا، ويدوّر عليها في قائمة سياراتك.
- تحديد موقع كل سيارة على خريطة حقيقية (اضغط، اسحب الدبوس، أو استخدم موقعك الحالي).
- واجهة عربية بالكامل (RTL) بتصميم يشبه اللوحات الرسمية.

## طريقة النشر على GitHub Pages

1. اعمل Repository جديد على GitHub (public).
2. ارفع ملف `index.html` الموجود في هذا المجلد لجذر الـ Repository (يعني في المسار الرئيسي مش جوه مجلد فرعي).
3. من إعدادات الريبو: **Settings → Pages**.
4. تحت **Build and deployment** اختار **Deploy from a branch**.
5. اختار البرانش `main` والمجلد `/ (root)`، ثم اضغط **Save**.
6. بعد دقيقة أو اتنين هيديك رابط شكله:
   `https://USERNAME.github.io/REPO-NAME/`

افتح الرابط من الموبايل أو الكمبيوتر وابدأ تستخدم التطبيق.

## ملاحظات مهمة
- **البحث الصوتي** يحتاج HTTPS (GitHub Pages بيوفره تلقائي) وإذن استخدام الميكروفون من المتصفح — الأنسب يكون Chrome.
- **البحث بالكاميرا (OCR)** يستخدم مكتبة Tesseract.js التي تعمل بالكامل داخل المتصفح (مفيش صور بترفع لأي سيرفر خارجي). أول مرة تستخدمها، المتصفح بيحمّل بيانات تدريب اللغة العربية (حوالي بضع ميجابايتات) من CDN، فلازم يكون فيه اتصال إنترنت في هذه اللحظة؛ بعدها بيتخزن مؤقتًا في المتصفح. يحتاج أيضًا إذن استخدام الكاميرا. النتيجة بتتحط في مربع البحث اليدوي تلقائيًا فتقدر تعدّلها لو القراءة مش مظبوطة ١٠٠٪.
- **البيانات** (السيارات اللي بتضيفها) بتتخزن مؤقتًا في الجلسة بس، ومش بتتحفظ بعد تحديث الصفحة، لأن التطبيق حاليًا مفيهوش قاعدة بيانات. لو عايز تخزين دائم، ده محتاج تعديل بسيط لإضافة تخزين خلفي.
- التطبيق مبني بملف HTML واحد فقط (بدون أي إعدادات بناء Build خاصة)، فمفيش أي خطوة تجهيز إضافية قبل الرفع.
